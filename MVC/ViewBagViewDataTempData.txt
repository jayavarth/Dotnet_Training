ViewBag, ViewData, TempData : are objects that are used to pass information/data from the controller to the view

ViewBag : Dynamic Data type 
    - used to transfer data from controller to view
    - has short life, meaning once the data is passed from the controller to the view, it becomes null
    - does not need any typecasting

Eg : ViewBag.Mydata = "Data One";

and to access the viewbag data in the view page
  @ViewBag.Mydata


ViewData : is a dictionary object that is derived from ViewDataDictionary
   - uses a key value pair for storing and retrieving data, where the key is always a string
   - used to transfer data from controller to view
   - has short life, meaning once the data is passed from the controller to the view, it becomes null
   - requires typecasting

Eg : ViewData["Mydata"] = "Data Two" ;

to access
   ViewData["Mydata"]


TempData : is a dictionary object that is derived from TempDataDictionary
    - Uses a key value pair
    - used to transfer data from controller to view/s
    - internally it makes use of session to store the values
    - always check for null before using the tempdata values to avoid runtime errors
    - requires typecasting
    - has bigger life than viewbag and viewdata
    - it is generally used to transfer data/values from one controller to another
    - Tempdata is able to carry the data to further request with the help of a method keep()


-- To pass data from the view to the controller
 1. Object (model)
 2. Forms Collection
 3. Parameter Collection of the model/entity
 4. Request Object

Http Verbs involved for any data transactions/crud operations
 - HttpGet : is a get request for a resource from the server
 - httpPost : is a post request for sending the data from the view to the controller and hence the server
 - HttpPut : is a request for change in data (from the view to the server)
 - HttpDelete : is a request for deletion of data
 - HttpPatch : selective updation request
